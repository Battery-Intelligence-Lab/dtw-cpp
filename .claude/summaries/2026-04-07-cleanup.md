# Session: Over-Engineering Cleanup (2026-04-07)

## What was done

Removed over-engineering introduced by previous sessions. Net: -795 lines, 29 files changed.

### Core changes
1. **DenseDistanceMatrix**: Replaced `std::vector<bool> computed_` with `-1.0` sentinel. No atomics, no locking — parallel fills use disjoint (i,j) pairs by design. `count_computed()`/`all_computed()` scan data (cold path only).
2. **Problem**: Deleted `is_distMat_filled` bool and `set_distance_matrix_filled()`. `isDistanceMatrixFilled()` delegates to `distMat.all_computed()`.
3. **Dead code**: Deleted `DistanceMetric` enum and `lb_pruning_compatible()`.
4. **Dispatch dedup**: Added 4 dispatch helpers (`dispatch_metric`, `dispatch_mv_metric`, `dispatch_missing_metric`, `dispatch_missing_mv_metric`), replaced 17 identical switch blocks.
5. **ScratchMatrix**: Changed public to private Eigen inheritance.
6. **MSVC `/fp:fast`**: Changed to `/fp:precise` + `/fp:contract` (preserves `std::isnan()`).
7. **Subproject guard**: Added target-level `-fno-finite-math-only` for GCC/Clang.
8. **Minor**: `band_bounds()` helper (5 sites), thread_local fixes, unused include removal.
9. **Stale files**: Deleted 6 report/summary files, condensed LESSONS.md.

## Test results
60/63 pass. 3 pre-existing failures: 2 CUDA (no GPU), 1 Benders (missing results/ dir).

## Decisions
- DTW distances are always >= 0, so -1.0 is a safe sentinel. Tests updated to remove negative-distance scenarios.
- `/fp:contract` requires MSVC 17.0+ (VS2022). Acceptable given project targets.
- AROW banded uses simpler band formula (`std::ceil` without round-100 trick) — left as-is with comment.
